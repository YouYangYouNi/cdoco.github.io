# onStart( mixed $hook )

Add a new hook to run in the context of the REPL when it starts.

Add a new hook to run in the context of the REPL when it starts.

Parameters

$hook
mixed
$hook The hook is either a string of PHP code to eval(), or a Closure accepting the EvalWorker object as its first argument and the array of defined local variables in the second argument. If the hook is a callback and needs to set any local variables in the REPL's scope, it should invoke $worker->setLocal($var_name, $value) to do so. Hooks are guaranteed to run in the order they were added and the state set by each hook is available to the next hook (either through global resources, such as classes and interfaces, or through the 2nd parameter of the callback, if any local variables were set.

Example

Contrived example where one hook sets the date and another prints it in the REPL. $boris->onStart(function($worker, $vars){ $worker->setLocal('date', date('Y-m-d')); }); $boris->onStart('echo "The date is $date\n";');
